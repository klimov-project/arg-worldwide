# Анализ платформы Argito.com

## 1. Что из себя представляет Argito?

**Argito** — это глобальная P2P (peer-to-peer) маркетплейс-платформа для размещения частных объявлений, аналогичная российскому Avito, но с международным охватом и поддержкой криптовалютных платежей.

### Текущий функционал (по данным сайта):

- Размещение объявлений о продаже товаров
- Геолокация продавцов (Аргентина, и др.)
- Мультивалютность (ARS, USD, RUB и другие)
- Категории товаров: электроника, одежда, авто, бытовая техника
- Галерея изображений товаров
- Ценообразование в местных валютах

## 2. Особенности платформы

### Уникальные черты:

| Особенность        | Описание                                               |
| ------------------ | ------------------------------------------------------ |
| Глобальный охват   | Объявления из разных стран (Аргентина видна в примере) |
| Мультивалютность   | Поддержка разных валют (ARS, USD, RUB)                 |
| Криптоплатежи      | Заявлена оплата в криптовалюте (требует уточнения)     |
| Минималистичный UI | Простой интерфейс без перегрузки                       |

### Стандартные элементы (аналогично Avito/OLX):

- Листинг товаров с фото и ценой
- Геолокация объявлений
- Категоризация товаров
- Контакт с продавцом

### Что отсутствует (по сравнению с современными маркетплейсами):

- **Trust Score / рейтинг доверия** — нет видимой системы
- **Верификация продавцов** — не обнаружена
- **Escrow-система** — безопасные сделки
- **Чат/мессенджер** — встроенная коммуникация
- **Мобильное приложение** — нет информации
- **Система отзывов** — не видна на главной

## 4. Технические вопросы для себя (как разработчику)

### Можно ли делать на Nuxt + SSR?

**Да, отлично подходит!**

| Преимущество       | Описание                                              |
| ------------------ | ----------------------------------------------------- |
| SEO                | SSR критичен для маркетплейса (индексация объявлений) |
| Производительность | Быстрая загрузка для пользователей                    |
| i18n               | Nuxt i18n для мультиязычности                         |
| API Routes         | Встроенные серверные эндпоинты                        |

**Рекомендуемый стек:**

```
Frontend: Nuxt 3 + Vue 3 + TypeScript
UI: Nuxt UI / Tailwind CSS
State: Pinia
API: Nuxt Server Routes + tRPC (опционально)
```

### Какой backend нужен?

**Минимальный MVP:**

- Node.js + Express/Fastify или Python + FastAPI
- PostgreSQL (основная БД)
- Redis (кэш, сессии, real-time)
- S3/Cloudflare R2 (изображения)

**Для масштабирования:**

- Elasticsearch (поиск объявлений)
- RabbitMQ/Kafka (очереди)
- Kubernetes (оркестрация)

**Для крипто:**

- Web3.js / Ethers.js (EVM)
- TON SDK (TON blockchain)
- Собственный или сторонний escrow смарт-контракт

### Интеграция кошельков

**Можно ли сделать авторизацию через кошелёк?**
**Да!** Это современный подход для Web3 проектов.

#### MetaMask (EVM chains):

```typescript
// Пример авторизации через MetaMask
import { ethers } from 'ethers';

async function connectMetaMask() {
  const provider = new ethers.BrowserProvider(window.ethereum);
  const signer = await provider.getSigner();
  const address = await signer.getAddress();

  // Sign message для верификации
  const message = `Login to Argito: ${Date.now()}`;
  const signature = await signer.signMessage(message);

  // Отправить на backend для проверки
  return { address, signature, message };
}
```

#### TonKeeper (TON):

```typescript
// Пример с TonConnect
import { TonConnect } from '@tonconnect/sdk';

const connector = new TonConnect({
  manifestUrl: 'https://argito.com/tonconnect-manifest.json',
});

async function connectTonKeeper() {
  const wallets = await connector.getWallets();
  await connector.connect({ jsBridgeKey: 'tonkeeper' });
}
```


```
Рекомендация:
```

```
- Продавец видит залоченные средства
- После подтверждения получения — release
- Dispute resolution через арбитраж
```

```

Рекомендация:
- Тёмная тема (популярна в крипто)
```

- Современного UX/UI

- UI компонентов (карточки, фильтры, дашборды)
- Trust Score системы
